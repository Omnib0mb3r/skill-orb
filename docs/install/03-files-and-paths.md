## Outside the repo (handled with care)

### `~/.claude/settings.json` — MODIFIED, BACKED UP

**Owner:** Claude Code itself. Edited by user, plugins, and tools (including DevNeural).

**What DevNeural does:**
- Adds 5 hook entries (PreToolUse, PostToolUse, UserPromptSubmit, Stop, Notification) pointing at `07-daemon/dist/capture/hooks/hook-runner.js` via `silent-runner.vbs`.
- The Notification entry forwards CC's permission/elicitation message to the daemon so the dashboard can render the question with answer buttons.
- Removes any v1 DevNeural hook entries (paths matching `01-data-layer/dist/hook-runner.js` or `04-session-intelligence/dist/session-start.js`).
- Leaves all OTHER hook entries intact (Claude-Setup hooks, plugin hooks, GSD hooks, your custom hooks, etc).
- A separate `silence-all-hooks.ps1` pass wraps every hook in `silent-shim.exe` with backslash-escaped (`\"`) inner args so spawns run hidden and survive bash invocation. Idempotent on re-run.

**Backup behavior:**
- Before any change, the file is copied to `~/.claude/settings.json.devneural.bak`.
- The backup is overwritten on each install run.

**Reset:** restore the `.bak` file to undo DevNeural's changes.

**Personalized inventory:** the exact list of hooks currently in your `settings.json` is not in this doc. **That inventory is captured in Phase 5 (settings audit).** See `docs/spec/phase-5-settings-audit.md`.

### `~/.claude/projects/` — READ ONLY

**Owner:** Claude Code. DevNeural never writes here.

**What DevNeural does:**
- Watches the directory tree with chokidar.
- Reads jsonl transcript files incrementally (persisted byte offset), never full-file reads.
- Stores byte offsets at `c:/dev/data/skill-connections/transcript-offsets.json`.

**Risk:** none. Read-only.

---

### `~/.claude/skills/` and `~/.claude/plugins/**/skills/` — READ ONLY

**Owner:** Claude Code and plugins.

**What DevNeural does:**
- Initial corpus ingest reads each `SKILL.md` to seed the wiki with skill knowledge.
- Re-read on `/devneural-reseed`.
- Never modified.

**Risk:** none. Read-only.

---

### `~/.claude/CLAUDE.md` (global) — READ ONLY

**Owner:** user (you). Backed by `Omnib0mb3r/Claude-Setup`.

**What DevNeural does:**
- Reads it as part of session context where applicable.
- Never modifies.

**Risk:** none.

## What DevNeural reads from your project repos

When the corpus seed or filesystem watcher runs, DevNeural reads (but does not modify) these files in any project under `C:/dev/Projects/<project>/`:

- `README.md`
- `CLAUDE.md`
- `devneural.jsonc`
- `OTLC-Brainstorm.MD`
- recent `git log` output (via `git log` shell call)

If a project has a `.devneural-ignore` file at its root, DevNeural skips capture and ingest for that project entirely.

## Files DevNeural will NEVER touch

For absolute clarity:

- `~/.claude/CLAUDE.md` (global user instructions)
- Any project's source code files
- Any project's `package.json`, `tsconfig.json`, etc.
- Your `.bashrc`, `.zshrc`, PowerShell profiles
- Windows registry
- Any file outside `C:/dev/data/skill-connections/`, `C:/dev/Projects/DevNeural/`, and the listed entries in `~/.claude/`

The only modification outside DevNeural's own directory is the surgical addition of hook entries to `~/.claude/settings.json`, with backup.
